Open Work Labs (2019)

At OWL, my ability to quickly pick up new skills and eye for design significantly increased team velocity. I created prototypes and established a visual identity for an early-stage company that was still searching for product-market-fit. Working with experienced product developers, I conducted user tests and designed products based on my findings. Under the tutelage of senior JavaScript engineers my code became far more maintainable and declarative. I also gained extensive experience working on in-production codebases built on top of experimental decentralized-web technologies.

  • ♒︎ Streams (source code):

    Streams is an experimental product that creates shareable feeds of your work; the protoype linked above organized GitHub events for any given repo. I was entirely responsible for the visual design and its implementation. I also contributed significant amounts of user reasearch and application logic (using react + redux).

  • ✵ Quasar (my commits):

    Quasar is a NodeJS server that allows file or dag pinning to IPFS nodes via (ethereum) smart contract events. I wrote significant parts of the application logic, and was personally responsible for docs and deployment setup.

  • ☾ OWL website (source code):

    I am responsible for all the design and code for the OWL site, including the logo. I also contributed significantly to the copy.

  • ⚙︎ Aragon

    Aragon is an organization building ethereum infrastructure that allows anyone to create decentralized organizations. I helped integrate Quasar with the Aragon frontend and contributed a few other various IPFS integrations.

Smart Museum (2018-19)

I used web and game development frameworks in concert to create engaging, novel interactive artworks for the web and gallery spaces. I produced a series of works that evolve over time in response to audience interactions.

UChicago Festival of the Arts (2018-19)

I collaborated with other artists to stage a series of audiovisual performances (the first project listed in link) using various combinations of Unity, livecoding GLSL, PureData, Ableton, and a web-connected audience. I created virtual and augmented reality sketches for informal showcase events on campus. These projects serve as better illustrations of my interests and ability to learn than my ability to code. I hacked them together until they functioned as I wanted, rather than taking care to write declarative, maintainable code (how I would approach them today).
